# Arcada Hotel

Концептуальний сайт готельно-ресторанного комплексу у Львові — портфоліо-проєкт на Astro та Tailwind CSS.

🔗 **Демо:** [ynkotp228.github.io/ArcadaHotel](https://ynkotp228.github.io/ArcadaHotel/) (автодеплой на GitHub Pages при кожному push у `main`)

---

## Короткий опис

Arcada Hotel — це вигаданий бренд, для якого реалізовано повноцінний маркетинговий сайт: головна сторінка, сторінка готелю з номерним фондом, ресторан, конференц-зали та окрема сторінка Design System з описом бренд-гайдлайну. Проєкт демонструє компонентну архітектуру на Astro, кастомну дизайн-систему на токенах Tailwind CSS та low-level інтерактивні елементи (карусель, off-canvas меню, форма з валідацією), написані без сторонніх UI-бібліотек.

Усі дані, ціни та контакти — вигадані; сайт створено виключно для демонстрації навичок верстки.

---

## Основні можливості

- **Кастомна дизайн-система**, зібрана на токенах Tailwind v4 (`@theme` у `global.css`) — окрема сторінка `/DesignSystem` документує палітру, типографіку, компоненти, форму елементів та голос бренду.
- **Off-canvas бургер-меню** з overlay, закриттям по кліку поза меню, по `Esc` та по кліку на посилання, з коректними `aria-expanded` / `aria-hidden` атрибутами.
- **Кастомна карусель** на чистому JavaScript (без бібліотек): стрілки, кліковні точки-індикатори та свайп для тач-екранів.
- **Форма зворотного зв'язку** з клієнтською валідацією в реальному часі (ім'я, телефон у форматі `+380XXXXXXXXX`, повідомлення), inline-помилками під полями та відправкою через Formspree без перезавантаження сторінки.
- **Google Maps** — вбудована інтерактивна карта з розташуванням готелю на сторінці контактів.
- **SEO-розмітка**: унікальні `title`/`description` для кожної сторінки, canonical URL, Open Graph та Twitter Card метатеги, автоматична генерація `sitemap.xml` через `@astrojs/sitemap`.
- **Оптимізовані зображення**: усі фото у форматі WebP, обробляються через `astro:assets` (`<Image />`) з `eager`/`fetchpriority="high"` для hero-зображень (LCP) та `lazy` — для другорядних.
- **Адаптивна верстка** для всіх сторінок і компонентів — від мобільного меню до перекомпоновки колонок і карток на менших екранах.
- **Автоматичний деплой** на GitHub Pages через GitHub Actions (`withastro/action`) при кожному push у `main`.

## Структура сторінок

| Сторінка | Маршрут | Призначення |
|---|---|---|
| **Головна** | `/` | Hero-екран, блок «Наша інфраструктура» (карусель Готель/Ресторан/Конференц-зали), «Про нас» та контакти з картою. |
| **Готель** | `/Hotel` | Hero-екран готелю та каталог із 4 типів номерів (Стандартний, Люкс, НапівЛюкс, Сімейний) з цінами та формою запиту. |
| **Ресторан** | `/Restaurant` | Hero-екран ресторану та три тематичні блоки (меню, святкування, гастрономічні майстер-класи) у форматі карток, що чергуються (зигзаг). |
| **Конференц-зали** | `/ConferenceHall` | Опис трьох залів різного формату — Smart, Optima, Maxi — у тому ж форматі карток-зигзаг. |
| **Design System** | `/DesignSystem` | Живий бренд-гайдлайн: палітра кольорів, типографічна шкала, зразки реальних компонентів, радіуси/тіні, тайминги анімацій і принципи голосу бренду. |
| **Незабаром** | `/soon` | Сторінка-заглушка для кнопок бронювання/замовлення з чесним поясненням, що це демонстраційний портфоліо-проєкт. |
| **404** | `/404` | Кастомна сторінка помилки з логотипом і кнопкою повернення на головну. |

## Особливості дизайну

- **Продумана колірна система**: три родини кольорів (navy, bronze, gold) плюс нейтральні cream/ink — усього 11 токенів, кожен із чітко визначеною роллю (заголовки, акценти дії, тло, текст).
- **Парна типографіка**: Playfair Display для заголовків (класичний «готельний» тон) і Inter для основного тексту (читабельність інтерфейсу).
- **Послідовна форма елементів**: заокруглення «драбинкою» — що більший блок, то м'якший радіус (`rounded-lg` → `rounded-2xl` → `rounded-3xl` → `rounded-full`), і єдиний стиль м'яких тіней з відтінком navy замість сірого.
- **Єдиний акцент дії**: бронзовий колір закріплений виключно за кнопками, роздільниками та активними станами — це явно задокументовано як правило («не змішувати bronze і gold в одному акценті»).
- **Фотографії, згенеровані штучним інтелектом** — зображення фасаду, номерів та інтер'єрів створені AI-генератором зображень, що дозволило зібрати цілісний візуальний ряд вигаданого бренду без використання стокових фото.

## UX-рішення

- Sticky-хедер із `backdrop-blur` — залишається читабельним поверх контенту під час скролу.
- Градієнтний оверлей (`navy` → прозорий) на всіх фонових зображеннях з текстом — гарантує контраст незалежно від фото.
- Ховер-мікровзаємодії: підйом кнопок і карток на `-2px`/`-4px`, плавне збільшення фото всередині картки при наведенні.
- Форма звернення дає миттєвий, конкретний фідбек: помилка з'являється під полем після втрати фокусу, а не лише при спробі відправки; поле телефону автопідставляє `+380` при фокусі.
- Чесна комунікація з користувачем: кнопки бронювання і замовлення ведуть на сторінку `/soon`, яка прямо пояснює, що це демо-проєкт, а не приховує непрацюючий функціонал.
- Дублювання ключових контактів (телефон, адреса) у хедері, бічному меню, блоці контактів і футері — користувач завжди на крок від зв'язку з готелем.

## Використані технології

- **[Astro](https://astro.build)** 6 — основний фреймворк, компонентна архітектура `.astro`
- **[Tailwind CSS](https://tailwindcss.com)** 4 (через `@tailwindcss/vite`) — utility-first стилізація та кастомні дизайн-токени (`@theme`)
- **TypeScript** — типізовані `Props`-інтерфейси компонентів, `tsconfig` у строгому режимі (`astro/tsconfigs/strict`)
- **`@astrojs/sitemap`** — автогенерація `sitemap.xml`
- **`astro:assets`** — оптимізація та обробка зображень (WebP)
- **Vanilla JavaScript** — карусель, бургер-меню, валідація форми (без сторонніх JS-бібліотек)
- **Formspree** — обробка відправки форми зворотного зв'язку
- **Google Fonts** (Playfair Display, Inter) та **Google Maps Embed**
- **GitHub Actions + GitHub Pages** — CI/CD для автоматичного деплою

## Що демонструє цей проєкт

- Побудову **компонентної архітектури** з перевикористанням: один компонент `Hero` обслуговує 4 різні сторінки через пропси, `Card`/`CardItem` — одразу дві сторінки з альтернативною (зигзаг) розкладкою.
- Уміння створювати та **документувати власну дизайн-систему**, а не просто «на око» підбирати кольори — токени, типографічна шкала й правила використання зафіксовані окремою сторінкою.
- Написання **інтерактивних елементів з нуля** на чистому JavaScript: карусель зі свайпом, доступне off-canvas меню, форма з покроковою клієнтською валідацією — без залежності від готових UI-kit рішень.
- Увагу до **продуктивності та SEO**: коректна пріоритизація завантаження зображень (LCP), метатеги Open Graph/Twitter, sitemap, семантична розмітка.
- Практики **доступності (a11y)**: ARIA-атрибути на інтерактивних елементах, керування фокусом, `aria-live` для статусів форми, підписи для декоративних та контентних зображень.
- Налаштування **CI/CD**: робочий пайплайн автодеплою на GitHub Pages через GitHub Actions.
- Написання **адаптивної верстки**, що коректно поводиться на мобільних, планшетних і десктопних екранах на всіх сторінках без винятку.

## Запуск проєкту локально

```bash
npm install
npm run dev       # локальний сервер розробки
npm run build     # продакшн-збірка у ./dist
npm run preview   # перегляд продакшн-збірки
```

Потрібен Node.js версії 22.12.0 або новіше.
